New Inbox

Note

Right now, only early adopters can use the new Inbox.

Inbox is your main page in the Medius Spend Management suite. It is divided into two parts:

  • A list of all tasks that you can display in default or custom views.

  • A preview of the selected task.

You can select a task from the list by clicking it. Details of the task will open on the right side. Each task contains tabs with information related to the given document type.

Inbox overview¶

The new Inbox offers several ways to view and manage invoices. These views help you process invoices faster by showing you which documents need your attention.

Using , you can create and save multiple custom views. Extensive filter options enable you to select the exact criteria you need. For example, you can filter for closed (invalidated and archived) invoices, incoming invoices, or filter by reason to check why the invoice stopped in Document Import.

Extended customization allows you to adjust row and column settings according to your needs:

  • Set the invoice counter to define how many invoice rows appear on each page. You can display up to 100 rows.

  • To change the order of columns, simply drag a selected column and drop it at the desired location.

  • To match the width of a column to its content, double click between the header of this column and the column on its right or left.

The system will remember your preferences once you save them.

Default views¶

There are two default views that show documents in separate tabs:

  • Assigned to you

  • All invoices

To switch between the views, simply click the name of the view you want to display.

Assigned to you¶

The Assigned to you view displays all invoices and documents directly assigned to you or related to your role. This view is linked to the Home page , where the counts of invoices assigned to you and to your role are shown on two separate tiles at the top of the page.

This section ensures that you focus on tasks specifically assigned to you, helping you manage your workload.

Tip

  • To see the full invoice, double-click the empty space in the invoice row.

  • Rows displayed in red indicate that the document includes errors.

  • Rows displayed in bold indicate that the document hasn’t been opened yet.

Additionally, you can add labels to tasks as described in the Labels section below.

All invoices¶

The All invoices view works like dashboard gadgets for searching invoices. It shows a full list of supplier invoices so you can see their status in detail. This view is linked to the Home page , where you can view all invoices in the workflow and their handlers.

This view offers extended filtering options. For example, it allows you to identify closed invoices (archived or invalidated).

Tip

In this view, you can filter invoices by current handler, the person the invoice is assigned to, and role. If you’re searching for a document that requires more detailed criteria, we recommend using the Reports functionality.

Filtering and custom views¶

In both Assigned to you and All invoices views, you can create custom views to show invoices based on your chosen criteria.

First, apply a filter to select what you want, then click Save as a new view. This makes it easier to find and process invoices in a way that suits you.

You can edit and reset filter settings to refine your custom views. The Reset filter changes option restores the latest saved view. If the view was not saved, it returns to the default setup. The Clear all filters option, on the other hand, removes all applied filters.

Tip

After you make your changes, remember to click Save filter changes to update your custom views.

The ellipsis button offers you options to rename or delete your custom views.

Tip

When filtering by company, press Space in the filter search box to display a list of all companies.

Example¶

Below you will find steps to follow when creating a custom view.

How to create a view showing only delegated non-PO invoices?¶

  1. Click and select Document type form the drop-down list.

  2. A Document type drop-down list will show. Select Non-PO invoice there and click Apply.

  3. Click again and select Indicators from the drop-down list.

  4. To display delegated invoices, select the Delegation indicator from the Indicators drop-down list and click Apply.

  5. Click Save and select Save as new view from the drop-down list to create a view showing delegated non-PO invoices.

  6. In the resultant pop-up window, type a name for your view.

  7. The view is available in a new tab.

Bulk operations¶

To process invoices in bulk, select the checkboxes next to the invoice rows. This will display a bar below the invoice list. Click the Bulk edit button on this bar to proceed with the bulk operation.

Bulk operations can only be done on documents of the same type. The following types of actions are available:

  • Workflow action – selected tasks must be the same to perform this action on a document. Available operations will be visible in the next step.

  • Update action – used to update information on documents. Fields that can be changed will be visible in the next step. Note that this action will not always be available.

  • Archive action – used to send documents straight to archive. Note that this action will not always be visible.

Note

Each item on the list represents a task related to a document. If a document has more than one task, it will be listed multiple times.

For example, if you belong to two or more roles responsible for handling different types of deviations, the same document will be listed once for each type of deviation.

Details panel¶

Details of the selected invoice appear on the right-hand side of Inbox. To view the invoice in full, double-click an empty space in its row.

The available tabs and options depend on the document type of the selected task. Each tab and option is described in the relevant sections of this documentation.

You can add a comment to each task. This comment will be visible to other users who receive the task in the next workflow steps.

The bottom right corner of the panel contains buttons that allow you to perform the following operations:

  • Save the changes applied to the document. Clicking save will not send the document to the next workflow step.

  • Send the document for review to users that you select. These users will receive the read access to the document.

  • Send the document to the next workflow step. When this is done, the next task will be selected in Inbox.

For invoices, the heading of the details panel contains a summary of important document details. An administrator can configure which fields appear in the summary in company configuration.

Column picker¶

Click the Columns settings button to decide which columns to display in each view. Only the columns you select here will be displayed. Once you click Save, the system will remember both the selected columns and their widths, and sequence.

Labels¶

You can add labels to documents to:

  • Display additional document information.

  • Filter the inbox to display only documents with the selected label.

You can add multiple labels to a document as long as the labels are available for the company. You can create and modify labels on the Label administration page.

Adding a label:

  • If there are no labels yet, click the button in the labels column and select the desired label.

  • If there are some labels already added, click the existing label and select another one from the list.

Removing a label:

  • Click the label in the Labels column. A list of all available labels will appear. Find the label you want to remove and deselect it.

Indicators¶

Indicators allow you to see at a glance if a document has any comments, attachments, cash discount opportunities or other specific features. Hover over an indicator icon displayed next to a document in Inbox to see more information.

In AP Automation, the following indicator types are available:

– informs you of comments added to the invoice. You can add comments to all types of invoices.

– informs you of attachments added to the invoice. You can add attachments to all types of invoices.

– for non-PO invoices in the Post Control step only, this additional indicator will be displayed if the invoice has any rejected lines.

– for PO invoices in the Post Control step only, this additional indicator will be displayed if the invoice has any rejected deviations (without adjustment request or short payment) or rejected invoice amount.

– marks PO invoices that have at least one connection to a PO line with a two-way matching policy.

– marks PO invoices connected to one or more purchase orders with at least one goods receipt missing.

– for contract based invoices only, this indicator will inform you that the invoice was generated automatically according to configuration on a contract.

– informs you of any active or inactive contracts in the Medius Contract module for the connected company and supplier. This indicator is visible:

  • only for non-PO and PO invoices

  • only if the user is assigned to the role Supplier Contract Application Access

  • regardless of whether the user has access to the contracts or not

– indicates an invoice for which a cash discount deadline is in the future. It helps you spot opportunities to save money by paying in accordance with cash discount terms.

– indicates a payment request that has a higher priority than other payment requests. Fast track payment requests appear at the top of the payment requests list in Inbox.

– informs you that a deviation has been rejected with short payment. It applies only to PO invoices with short payment configured.

– notifies you of active risk factors associated with this invoice. This indicator can be displayed for documents at various workflow steps.

– informs you that risk factors associated with this invoice have been handled. This indicator can be displayed on documents at various workflow steps.

– informs you that the invoice has been delegated to another user.

– informs you that the invoice has been escalated.

– informs you that additional coding is required, or the confidence level of coding suggestions in some segments is not high enough.

– informs you that a coding suggestion is available or applied to all the coding segments with a high confidence level.

– informs you that accrual has been applied to at least one of the invoice lines.

Error handling¶

Validation of a task is done in the background. This means you can move on to other tasks before the validation is complete. If a validation error occurs, the task is marked in red color in Inbox.

When a document has an error, all changes that were made to it will be saved. To restore the document to its original state, click the button. To handle the error:

  • Open the task by clicking the invoice row in Inbox.

  • Correct the error.

    Note

    The majority of errors displayed in Inbox are related to Validator Configuration. We suggest you check your validation settings in search of the problem resolution.

  • Resend the document.
